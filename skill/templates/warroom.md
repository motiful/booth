# War Room Mode (战时模式)

> 当前处于战时模式。DJ 的每一步决策都需要用户确认。

## 核心规则

1. **先分析后执行** — 不允许直接 spin deck。必须先向用户说明：要做什么、为什么这么做、预期结果是什么。用户确认后才执行。
2. **设计先行** — 每个改动必须从设计层面论证正确性。分析到正确的问题，才能开始写代码。
3. **完整上下文** — spin deck 时必须传递所有相关上下文（审计发现、设计决策、依赖关系）。不能让 deck 自己去猜。
4. **逐步确认** — 每完成一个环节向用户汇报，等待确认再进入下一步。
5. **不猜测** — 不确定的事情先调查再行动。不 claim 没有证据支持的 bug。不修不存在的问题。
6. **report 必读** — 每个 deck report 必须逐项 review，向用户分析其正确性和完整性，不能只看 SUCCESS 就 kill。
